# SwapIt 보상 크레딧 산정 로직 v2.1

> 최종 업데이트: 2026-06-10
> 반영 내용: 캡 정책 방안 1 적용 — 스크랩 100% 보장 + 보너스에만 10% 상한 적용 / 이용 횟수 기준 확정 — 크레딧 발급 완료 시점 / 신제품 등급 분류 확정 — 가격 구간 자동 분류

---

## 1. 핵심 개념

반납 가전의 **원자재 스크랩 가치(Floor)** 를 100% 보장하고,
신제품 구매 시 등급·이용 횟수에 따른 **보너스 크레딧**을 별도로 추가하는 하이브리드 보상 구조.

보상은 두 가지 형태로 지급된다.
- **결제 크레딧** — 신제품 구매 즉시 차감
- **LG Best Care AMC 골드 플랜** — 이용 횟수 연동 무상 제공 (별도)

---

## 2. 최종 보상 공식 (v3.0 확정)

```
최종 크레딧 = 스크랩 가치 + 신제품가 × 등급별 비율
```

### 핵심 원칙

| 항목 | 정책 |
|------|------|
| 스크랩 가치 | **무조건 100% 지급** |
| 보너스 | 신제품가 × 등급별 비율 (하한 없음, **상한 캡 없음**) |

> **v2.1 → v3.0 변경 이유**
> 기존에는 `min(신제품가 × 비율, 신제품가 × 10%)` 캡이 있었으나 제거.
> 등급별 비율이 그대로 적용되어 고객이 받는 혜택이 더 명확하고 크게 전달됨.

## 4. 크레딧 비율 매트릭스 (보너스 기본 비율)

구매 신제품 등급(변수 A)과 누적 이용 횟수(변수 B)의 교차값.

### 신제품 등급 분류 기준 (확정)

가격 구간으로 자동 분류. `market_products.price` 기준으로 서버가 판단.

| 등급 | 가격 구간 | 예시 |
|------|---------|------|
| **프리미엄** | 150만원 이상 | 오브제 컬렉션, LG SIGNATURE |
| **일반** | 50만원 ~ 150만원 미만 | 디오스/트롬/휘센 일반형 |
| **보급형** | 50만원 미만 | 엔트리 모델 |

> LG 내부 DB 연동 시 라인명(오브제, SIGNATURE 등) 기준으로 고도화 가능.

### 크레딧 비율 매트릭스

| 신제품 등급 | 1회 (최초) | 2회 (실버) | 3회 이상 (VIP) |
|------------|-----------|-----------|--------------|
| **프리미엄** (150만원 이상) | 10% | 12% | 15% |
| **일반** (50~150만원) | 7% | 10% | 12% |
| **보급형** (50만원 미만) | 4% | 7% | 9% |

> v3.0부터 `min()` 캡 제거. 보너스 실지급 = `신제품가 × 등급별 비율` (상한 없음).

## 5. AMC 락인 혜택 패키지

SwapIt 교체 고객 전원에게 결제 크레딧과 **별개로** 제공.

| 이용 횟수 | AMC 골드 무상 기간 |
|----------|-----------------|
| 1회 | 12개월 |
| 2회 | 18개월 |
| 3회 이상 | 24개월 |

포함 내용: 방문 공임비 + 정품 부품비 + 냉매 가스 충전 100% 무상, 연 1회 예방 점검.

## 6. 계산 예시

### 케이스 A — 중형 냉장고 반납, 프리미엄 신제품 구매, 최초 이용

```
스크랩:  냉장고 중형  →  39,000원
신제품:  100만원 프리미엄, 1회  →  비율 10%

보너스:  min(1,000,000 × 10%,  1,000,000 × 10%)
       = min(100,000,  100,000)  =  100,000원

최종 크레딧:  39,000 + 100,000  =  139,000원 (13.9만원)
AMC:  12개월 골드 무상
```

### 케이스 B — 대형 냉장고 반납, 보급형 신제품 구매, 최초 이용 (구 방식 충돌 케이스)

```
스크랩:  냉장고 대형  →  55,000원
신제품:  30만원 보급형, 1회  →  비율 4%

보너스:  min(300,000 × 4%,  300,000 × 10%)
       = min(12,000,  30,000)  =  12,000원

최종 크레딧:  55,000 + 12,000  =  67,000원 (6.7만원)  ✅ 스크랩 전액 보장
AMC:  12개월 골드 무상

※ 구 방식(전체 캡 15%) 적용 시:
   min(55,000 + 12,000,  300,000 × 15%) = min(67,000, 45,000) = 45,000원  ❌ 스크랩 손실
```

### 케이스 C — 세탁기 반납, 보급형 세탁기 구매, VIP (3회)

```
스크랩:  세탁기 중형  →  45,000원
신제품:  30만원 보급형, 3회  →  비율 9%

보너스:  min(300,000 × 9%,  300,000 × 10%)
       = min(27,000,  30,000)  =  27,000원

최종 크레딧:  45,000 + 27,000  =  72,000원 (7.2만원)
AMC:  24개월 골드 무상 (VIP 우대)
```

## 7. VLM이 가격 산정을 위해 반환해야 하는 정보

가격 산정 API 호출 시 VLM 분석 결과에서 아래 필드가 필요하다.

| 필드 | 용도 | 예시 |
|------|------|------|
| `applianceType` | 스크랩 테이블 조회 | `"냉장고"` |
| `size` | 스크랩 테이블 조회 | `"대형"` |
| `conditionGrade` | 현재 스크랩에 미반영, 향후 감가 적용 고려 | `"양호"` |

신제품 정보 및 이용 횟수는 VLM이 아닌 서버에서 별도 조회.

| 필드 | 출처 |
|------|------|
| 신제품 가격 | 상품 카탈로그 DB (`market_products.price`) |
| 신제품 등급 (프리미엄/일반/보급형) | 상품 카탈로그 DB (`market_products.grade` — 추가 필요) |
| 누적 이용 횟수 | 유저 DB (`users.swap_count` — 추가 필요) |

### 이용 횟수 카운트 기준 (확정)

**크레딧 발급 완료 시점** (`credits.status = ISSUED` 처리 완료 시) 에 `users.swap_count += 1`

| 상황 | 카운트 여부 |
|------|-----------|
| 촬영만 하고 이탈 | ❌ 미카운트 |
| 수거 신청 후 취소 | ❌ 미카운트 |
| 수거 완료됐으나 최종 검수 탈락 | ❌ 미카운트 |
| 크레딧 발급까지 완주 | ✅ +1 카운트 |

> 교체 구매까지 완주한 고객에게만 다음 회차 혜택이 쌓이는 구조.
> DB에 `users.swap_count INT DEFAULT 0` 컬럼 추가 필요 (현재 DB객체정의서 v2에 미포함 → 반영 요청 필요).

## 9. VLM 고도화 로드맵 (향후 과제)

### 현재 구현 방식
GPT-4o Vision이 사진 한 장으로 모델명 인식 + 해당 모델 스펙(용량/크기)까지 한 번에 판단.
모델명을 확실히 인식한 경우 `sizeSource: "model_spec"`, 불확실한 경우 `sizeSource: "visual_estimate"`로 구분.

### 향후 디벨롭 방향 (선택지)

**방향 A — 제품 스펙 DB 구축 + GPT 폴백**
```
모델명 추출 → 스펙 DB 조회 → 없으면 GPT 판단
```
- 장점: 할루시네이션 없이 정확한 스펙 보장
- 필요한 것: 주요 LG 모델 스펙 DB 구축 또는 에너지공단 공공 API 연동

**방향 B — LG 내부 제품 카탈로그 DB 직접 연동**
- 장점: 가장 정확, 신제품도 즉시 반영
- 필요한 것: LG 내부 시스템 연동 협의

**방향 C — 파인튜닝 모델**
- LG 가전 이미지 + 스펙 데이터로 별도 모델 학습
- 장점: 빠르고 정확, API 비용 절감
- 필요한 것: 학습 데이터 수집 및 파인튜닝 비용

## 10. 스크랩 가치 고도화 로드맵 (향후 과제)

현재는 고정 테이블(수동) 방식으로 운영. 아래 순서로 고도화 예정.

### 1단계 — 외부 API를 통한 가격 자동 산정
원자재 시세 API 또는 폐가전 실거래 데이터 API를 활용해 분기마다 수동으로 업데이트하던 테이블을 자동화.

```
분기 1회 배치 스케줄러 실행
→ 외부 API에서 원자재 시세 또는 실거래 가격 조회
→ 스크랩 테이블 자동 업데이트
```

### 2단계 — 머신러닝 모델을 통한 정밀 산정 (데이터 충분히 쌓인 후)
SwapIt 운영 데이터가 쌓이면 ML 모델로 외관 상태까지 반영한 정밀 산정 가능.

```
입력: 제품 종류, 크기, 연식, 외관 상태(VLM), 브랜드
출력: 예상 스크랩 가치 + 신뢰 구간
```
